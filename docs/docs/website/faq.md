# FAQ

This document serves to answer some common questions about the project.

## How do I get started?

It's recommended to read the [introduction](./intro) document.
This goes over the basics of the project and how to
get your environment set up. Once you read the necessary information,
look at the GitHub issues tracker to see what features or issues
needs to be tackled at. If you assign yourself to deal with an issue,
**please ensure that you remain active and committed**. There is
no strict deadline for when the feature is due unless noted. If you have
any questions, feel free to put them on the Discord server under the
`ucmacm-website` channel.

## How do I contribute?

Please read the [contributing](./contributing) document.
This document goes over the basics of how to contribute
and the suggested guidelines to follow.
